# easyNote 开发计划

> **给执行代理的要求：** 实施本计划时，建议使用 `superpowers:subagent-driven-development`，也可以使用 `superpowers:executing-plans` 按任务推进。任务使用 checkbox（`- [ ]`）格式记录进度。

**目标：** 构建 easyNote 的 Windows 桌面端第一版，实现本地工作区、置顶悬浮入口、任务增删改查、图片附件、导入导出等核心能力。

**架构：** Electron main process 负责窗口、文件系统、工作区、附件、导入导出等本地能力；preload 通过类型化 `window.easyNoteApi` 暴露受控 API；React renderer 只消费 API 和前端状态，不直接访问文件系统。第一版使用 JSON + 附件目录存储，但通过 Repository / Service 边界为后续 SQLite 或团队同步预留替换点。

**技术栈：** React 18、Electron、TypeScript、Vite、Zustand、lucide-react、adm-zip、nanoid。

```ts
export class TaskRepository {
  list(): Promise<Task[]>
  create(input: TaskInput): Promise<Task>
  update(id: string, input: Partial<TaskInput>): Promise<Task>
  remove(id: string): Promise<void>
  setStatus(id: string, status: TaskStatus): Promise<Task>
}
```

规则：

- `list` 按 `dueDate` 升序、`createdAt` 降序排序。
- `create` 对 `title` 做 trim，空标题直接拒绝。
- `create` 使用传入状态，通常默认为 `todo`。
- `update` 更新 `updatedAt`。
- `setStatus('done')` 写入 `completedAt`。
- 从 `done` 切换回其他状态时清空 `completedAt`。
- `remove` 只删除任务记录；附件删除由后续任务统一接入。
- 每次变更都递增 `revision`。
